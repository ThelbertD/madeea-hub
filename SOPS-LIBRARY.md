# MadeEA — Working SOPs Library

These are the **working SOPs** currently built into the Command Center. Each one
is a guided checklist an EA ticks through, with clear **Success Criteria** so it's
obvious when the task is done.

You've already seen **Inbox Triage** (included at the end for reference). Below are
the **other SOPs** we've generated — please review and suggest changes, or propose
new ones using the template at the bottom.

*(Steps marked "optional" don't block completion; everything else must be ticked before an SOP can be marked complete.)*

> **📌 Pin-to-screen checklist.** Any SOP can be "pinned" as a small floating
> checklist that stays on screen while the EA works — they tick items off as they
> go (in their inbox, calendar, anywhere in the app), and progress saves
> automatically. It can be collapsed to a tab or closed when finished. This keeps
> the SOP *with* the EA during the task instead of buried in a document — the whole
> point of "working SOPs."

## 1. Meeting Preparation
**Use when:** preparing an executive for an upcoming meeting.

**Checklist**
- ☐ Attendee profiles compiled
- ☐ Agenda drafted
- ☐ Relevant documents attached
- ☐ AI meeting brief generated *(optional — one click)*
- ☐ Pre-meeting reminder sent to participants

**Success Criteria / Deliverables**
- ✓ Meeting brief delivered to the executive
- ✓ Reminders sent to all participants
- ✓ Documents shared

## 2. Executive Priority Alignment
**Use when:** producing the executive's prioritised daily briefing (start of day).

**Checklist**
- ☐ Calendar reviewed
- ☐ Emails and tasks reviewed
- ☐ Conflicts and urgent items flagged
- ☐ Daily brief generated
- ☐ Brief delivered to the executive

**Success Criteria / Deliverables**
- ✓ Prioritised daily brief delivered before 8 AM
- ✓ Conflicts surfaced and resolved or flagged

## 3. Expense & Bookkeeping
**Use when:** compiling and submitting an expense report.

**Checklist**
- ☐ Receipts collected
- ☐ Expenses categorised
- ☐ Expense report generated *(AI-assisted — one click)*
- ☐ Submitted for approval
- ☐ Logged / filed in the system

**Success Criteria / Deliverables**
- ✓ Expense report generated
- ✓ Submitted to finance
- ✓ Logged and filed

## 4. Client Onboarding
**Use when:** bringing a new client into the system and kicking off the relationship.

**Checklist**
- ☐ Discovery call completed
- ☐ Client profile created in Client Vault
- ☐ Communication preferences captured
- ☐ Kickoff / welcome sent
- ☐ 30-day check-in scheduled

**Success Criteria / Deliverables**
- ✓ Client profile complete in the Vault
- ✓ Welcome / kickoff delivered
- ✓ Check-in on the calendar

## (Reference) Inbox Triage — the one already reviewed
**Use when:** handling an incoming request to standard.

**Checklist**
- ☐ Request acknowledged within 15 minutes
- ☐ Urgency assigned (Urgent / Standard / Low)
- ☐ Client profile reviewed
- ☐ AI support used *(optional)*
- ☐ Output reviewed against quality standards
- ☐ Response / action delivered
- ☐ Client preferences updated *(optional)*

**Success Criteria / Deliverables**
- ✓ Client has received a response
- ✓ Required action has been completed
- ✓ CRM / system has been updated
- ✓ Client profile updated with any new preferences
